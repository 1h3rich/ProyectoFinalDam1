GestionBaseDeDatos — MÉTODOS PÚBLICOS
----------------------------------------------------------------

vincularBDD()
  Conecta Java con la BD MySQL. Se llama al iniciar la aplicación.
  Ejecuta el DDL de creación de tablas (silencia errores si ya existen) y,
  si la tabla ciclo está vacía, inserta los datos por defecto mediante
  insertarDatosSiVacia() para evitar duplicados en cada arranque.

cerrarBDD()
  Cierra la conexión.

realizarConsultaSQL(String[] datosConsulta, String[] entradas, boolean mostrar, boolean guardar)
  Ejecuta un SELECT.
  Si guardar=false: datosConsulta[0]=SQL, datosConsulta[1..n]=columnas a mostrar.
  Si guardar=true:  datosConsulta[0]=tipo ("Alumno"...), datosConsulta[1]=SQL, datosConsulta[2..n]=columnas.

insertarDatos(String[] datosInsertar, String[] entradas)
  Ejecuta un INSERT. datosInsertar[0]=tipo, datosInsertar[1]=SQL.
  Ejemplo: GestionBaseDeDatos.insertarDatos(ConsultasSQL.INSERT_ALUMNO, entradas)

actualizarFila(String[] datosActualizacion, String[] entradas)
  Ejecuta un UPDATE. datosActualizacion[0]=SQL.
  Ejemplo: GestionBaseDeDatos.actualizarFila(ConsultasSQL.UPDATE_ALUMNO, entradas)

eliminarFila(String[] datosEliminar, String[] entradas)
  Ejecuta un DELETE. datosEliminar[0]=SQL.

obtenerUltimoCodigo(String tabla)
  Devuelve el MAX(codigo) de la tabla indicada. Lo usan los constructores de los modelos.

obtenerTableModel(String sql, String[] entradas)
  Devuelve un DefaultTableModel con los resultados del SELECT. Lo usan las pantallas de consulta.

insertarYDevolverID(String sql, String[] params)
  INSERT con AUTO_INCREMENT; devuelve el ID generado, o -1 si falla.

insertarSinID(String sql, String[] params)
  INSERT sin AUTO_INCREMENT (tablas con PK compuesta, o imports con código explícito).
  Devuelve true si se insertó al menos una fila.
  Usado por Exportar e Importar2 para el paso de sincronización con BD:
    → intenta INSERT; si falla (registro ya existe), se llama a actualizarFila.

PATRÓN UPSERT usado en Exportar (sin soporte nativo de MySQL UPSERT):
  boolean ok = GestionBaseDeDatos.insertarSinID(ConsultasSQL.INSERT_*_CON_CODIGO[1], paramsInsert);
  if (!ok) GestionBaseDeDatos.actualizarFila(ConsultasSQL.UPDATE_*, paramsUpdate);


----------------------------------------------------------------
ConsultasSQL — CONSTANTES SQL DISPONIBLES
----------------------------------------------------------------

INSERT:  INSERT_ALUMNO, INSERT_CICLO, INSERT_MODULO, INSERT_MATRICULA, INSERT_LINEA_MATRICULA
UPDATE:  UPDATE_ALUMNO, UPDATE_CICLO, UPDATE_MODULO, UPDATE_MATRICULA, UPDATE_LINEA_MATRICULA
DELETE:  DELETE_ALUMNO, DELETE_CICLO, DELETE_MODULO, DELETE_MATRICULA, DELETE_LINEA_MATRICULA
SELECT pantalla: SELECT_ALUMNO_TODOS, SELECT_CICLO_TODOS, SELECT_MODULO_TODOS...
SELECT objeto:   SAVE_ALUMNO_TODOS, SAVE_CICLO_TODOS, SAVE_MODULO_TODOS...
SELECT por id:   SELECT_ALUMNO_POR_CODIGO, SELECT_CICLO_POR_CODIGO...

Orden de parámetros UPDATE (para el array entradas[]):
  UPDATE_ALUMNO:          nombre, fecha_nacimiento, domicilio, telefono, correo, codigo
  UPDATE_CICLO:           denominacion, familia_profesional, nivel, horas, anio_curriculo, codigo
  UPDATE_MODULO:          codigo_ciclo, nombre, curso, creditos_ects, horas, codigo
  UPDATE_MATRICULA:       codigo_alumno, anio_academico, estado, importe, codigo
  UPDATE_LINEA_MATRICULA: repeticion, calificacion_primera, calificacion_segunda, codigo_matricula, codigo_modulo

----------------------------------------------------------------
PATRÓN DE USO — CÓMO ABRIR UN FORMULARIO MODIFICAR
----------------------------------------------------------------

Desde el menú correspondiente, al elegir "Modificar", se hace:

  Alumno seleccionado = ...;  // obtener de SesionDatos o de la tabla
  new ModificarAlumno(ModoFormulario.MODIFICAR, seleccionado).setVisible(true);

  // Para crear uno nuevo:
  new ModificarAlumno(ModoFormulario.CREAR, null).setVisible(true);

El mismo patrón aplica a ModificarCiclo, ModificarModulo, ModificarMatricula
y ModificarLineaMatricula.


----------------------------------------------------------------
BASE DE DATOS — centro_formacion
----------------------------------------------------------------

Tablas:
  ciclo          (codigo PK, denominacion, familia_profesional, nivel, horas, anio_curriculo)
  modulo         (codigo PK, nombre, curso, horas, creditos_ects, codigo_ciclo FK→ciclo)
  alumno         (codigo PK, nombre, correo UNIQUE, domicilio, telefono, fecha_nacimiento)
  matricula      (codigo PK, estado, importe, anio_academico, codigo_alumno FK→alumno)
  linea_matricula(codigo_matricula FK, codigo_modulo FK, calificacion_primera, calificacion_segunda, repeticion INT)
                  PK compuesta: (codigo_matricula, codigo_modulo)
                  repeticion: número de veces que el alumno ha cursado el módulo (0 = primera vez)

Conexión:
  URL:      jdbc:mysql://127.0.0.1:3306/centro_formacion
  Usuario:  root
  Contraseña: alumno (o root como fallback)
