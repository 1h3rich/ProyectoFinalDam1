CRISS

Centro de Recursos e Información para Sistemas de Servicios

Java 21 Swing MySQL Maven DAM1 · 2026

CRISS es un sistema de gestión integral para un Centro de Formación Profesional. Permite administrar el ciclo de vida completo de las entidades educativas —alumnos, ciclos formativos, módulos, matrículas y líneas de matrícula— tanto desde una interfaz de consola interactiva como desde una interfaz gráfica Swing.

La aplicación combina persistencia en base de datos MySQL con soporte de importación/exportación a múltiples formatos de fichero (CSV, JSON, TXT, Binario), procesamiento concurrente con SwingWorker, y un sistema de sesión en memoria basado en colecciones Java.

Características

👥 Gestión de Alumnos

  • Alta, baja y modificación de alumnos
  • Validación de teléfono, correo y fecha
  • Consulta individual y global
  • Importación/exportación multiformato

🏫 Ciclos y Módulos

  • Gestión de ciclos formativos y familias
  • Módulos con créditos ECTS y horas
  • Eliminación en cascada
  • Gestión gráfica de módulos del ciclo

📄 Matrículas

  • Matrículas por año académico
  • Líneas de matrícula con calificaciones
  • Control de repetición de módulo
  • Consulta de matrículas del alumno

💾 Persistencia

  • Base de datos MySQL con transacciones
  • Sesión en memoria (TreeSet / ArrayList)
  • Exportación a CSV, JSON, TXT, Binario
  • Importación concurrente (SwingWorker)

🔍 Consultas SQL

  • 9 ejercicios de consulta multitabla
  • Visor de resultados en tabla gráfica
  • Consulta filtrada por parámetro
  • Consulta completa por entidad

⚙ Infraestructura

  • 22 excepciones de dominio personalizadas
  • Validadores reutilizables
  • Registro de sesión auditado
  • Pantalla de créditos y no-click

Paquetes

Descripción de paquetes del proyecto CRISS
Paquete Contenido Clases
modelos Entidades de dominio: Alumno, Ciclo, Modulo, Matricula, LineaMatricula. Implementan InterpolaridadDeDatos, Serializable y Comparable<T>. 5
interfaces Contrato de serialización multiformato: InterpolaridadDeDatos define 11 métodos de exportación e importación (CSV, JSON, TXT, Binario). 1
servicios GestionBaseDeDatos (JDBC + transacciones), SesionDatos (caché Singleton), GestionFicheros (I/O), ConsultasSQL y ConsultasEjercicios. 5
Menus Menús de consola interactivos para cada entidad del sistema y para las consultas. Gestionan el flujo de entrada del usuario por teclado. 6
pantallas Interfaces gráficas Swing: creación, modificación, eliminación, consulta, concurrencia (importación/exportación), registro de sesión y créditos. 28
Control Controlador de interfaz gráfica que actúa como hub de navegación entre pantallas Swing. 1
excepciones 22 excepciones de dominio personalizadas (AlumnoVacioException, FechaNoValidaException, TelefonoInvalidoException, etc.) para validación de entidades. 22
Utils Validadores con métodos de validación reutilizables, ItemCombo para JComboBox, TipoDato y GsonUtils para serialización JSON. 4
Config Clase de constantes estáticas: cadena de conexión MySQL, credenciales y rutas de ficheros de exportación/importación por entidad. 1
Main Punto de entrada. Inicializa la conexión a base de datos, carga los datos en sesión y lanza el menú principal o la interfaz gráfica. 1

Tecnologías

Java 21

  • Collections: TreeSet, ArrayList, LinkedHashMap
  • Concurrencia: SwingWorker, ExecutorService
  • java.io / java.nio para ficheros
  • Generics y Comparable<T>

Swing (javax.swing)

  • JFrame, JPanel, JTable, JComboBox
  • DefaultTableModel para resultados SQL
  • SwingUtilities.invokeLater
  • Diseñado con NetBeans GUI Builder

MySQL + JDBC

  • Conector MySQL JDBC 8.x
  • Transacciones (commit/rollback)
  • PreparedStatement contra SQL injection
  • Esquema: alumnos, ciclos, modulos, matriculas

Maven

  • maven-compiler-plugin (Java 21)
  • maven-shade-plugin (fat JAR ejecutable)
  • maven-javadoc-plugin 3.12
  • Gson 2.x para serialización JSON
